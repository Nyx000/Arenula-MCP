Architecture

AI Client ──> HTTP/SSE ──> ArenulaMcpServer (inside s&box editor) │ RpcDispatcher │ ┌───────────────┼───────────────┐ │ │ │ async tools console path main thread (compile, cloud) (exception (all other background isolation) scene tools) │ │ │ Handler Handler Handler

28 C# files: 8 Core infrastructure + 20 Handlers (one per tool). Async tools dispatch before GameTask.MainThread() to avoid deadlocks.

Design Principles

Following Anthropic's official MCP guidance.

Omnibus Tools

Related operations grouped with action enum. Fewer tools = less selection ambiguity for the LLM.

Rich Descriptions

3-4 sentences per tool. What it does, when to use it, when NOT to use it, what actions are available.

Trimmed Responses

Canonical {id, name} tuples. Pagination with offset/limit. 16K char truncation. Concise/detailed modes.

Actionable Errors

Structured {error, action, suggestion} with fuzzy type matching. "Did you mean 'Rigidbody'?"

Schema Safety

additionalProperties: false on every schema. Enum constraints on action parameters. Prevents hallucination.

Async Safety

Network tools (compile, cloud) dispatch before MainThread. Never .GetAwaiter().GetResult() on main thread.

Installation

1. Copy the Editor plugin

# Copy into your s&box project's Libraries folder Arenula-MCP/editor/ → YourProject/Libraries/arenula_mcp/

2. Build the API and Docs servers

# API server cd Arenula-MCP/api && npm install && npm run build # Docs server cd Arenula-MCP/docs && npm install && npm run build

3. Configure your .mcp.json

{ "mcpServers": { "editor": { "type": "sse", "url": "http://localhost:8098/sse" }, "api": { "command": "node", "args": ["path/to/Arenula-MCP/api/dist/index.js"] }, "docs": { "command": "node", "args": ["path/to/Arenula-MCP/docs/dist/index.js"] } } }

4. Open s&box

Arenula Editor compiles automatically and starts the MCP server on port 8098. Tools appear as mcp__editor__*, mcp__api__*, mcp__docs__*.
